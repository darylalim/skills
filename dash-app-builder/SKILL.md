# Dash App Builder

Scaffold a single-file Dash app that loads a Hugging Face tabular dataset, lets the user filter every column, and shows a preview table plus one auto-chosen Plotly chart.

## Step 1: Identify the input

The skill accepts exactly one input shape: a Hugging Face dataset URL.

`https://huggingface.co/datasets/<org>/<dataset>` (with or without trailing slash; query strings and fragments are stripped before classification).

**Reject any other input shape** with this exact message:

> This skill builds Dash apps from Hugging Face dataset cards (`huggingface.co/datasets/<org>/<dataset>`). For scripts, notebooks, or other inputs, use a general-purpose Dash app prompt without this skill.

This applies to: model card URLs (`huggingface.co/<org>/<model>` with no `datasets/` segment), GitHub URLs, `.ipynb` URLs, file paths, and anything else that doesn't match the dataset URL shape.

## Step 2: Classify and route

Hybrid detection: metadata-first, schema-fallback.

1. **Fetch metadata** with `huggingface_hub.dataset_info(<org>/<dataset>)`. Capture `task_categories` and `gated`.

2. **Reject early on clearly-non-tabular tags.** If `task_categories` overlaps with this set, fail with the wrong-modality message:

```python
NON_TABULAR_TASK_TAGS = {
    "image-classification", "image-segmentation", "object-detection",
    "image-to-image", "image-to-text", "image-text-to-text", "text-to-image",
    "automatic-speech-recognition", "text-to-speech", "audio-classification",
    "audio-to-audio", "voice-activity-detection",
    "video-classification", "text-to-video",
    "depth-estimation", "image-feature-extraction",
}
```

3. **Accept on tabular tags.** If `task_categories` overlaps with this set, accept and skip schema inspection:

```python
TABULAR_TASK_TAGS = {
    "tabular-classification",
    "tabular-regression",
    "time-series-forecasting",
    "tabular-to-text",
}
```

4. **Otherwise, inspect the schema.** Call `datasets.load_dataset(<dataset_id>, split="train", streaming=True)` and check `.features`. Tabular if every top-level feature is `Value` (`int*`, `float*`, `string`, `bool`) or `ClassLabel`. Reject if any `Image`, `Audio`, `Sequence`, `Array2D`/`Array3D`, or nested `Features` is present.

The wrong-modality rejection message:

> This skill builds Dash apps from **tabular** Hugging Face datasets. The dataset `<dataset_id>` contains <detected_modality> data, which isn't supported.

Substitute `<dataset_id>` with the user's input and `<detected_modality>` with one of: `image`, `audio`, `video`, `sequence` (for `Sequence` features), `nested` (for `Features` substructs), or the offending tag (for the metadata path).

## Step 3: Scaffold

Produce four files. All column-aware logic (filter widget selection, chart type pick) is **runtime introspection** in T2/T3 — the skill does not need to inspect the schema beyond Step 2's reject decision.

### File 1: `dash_app.py`

Assemble templates **T1 + T2 + T3 + T4** in order from `references/scaffolding-templates.md`. Substitute `DATASET_ID = "<org>/<dataset>"` with the user's slug. The gated-gate block (the `if not HF_TOKEN: raise SystemExit(...)` lines in T1) is **emitted only when** `dataset_info.gated` is truthy.

### File 2: `test_dash_app.py`

Use template **T5** verbatim from `references/scaffolding-templates.md`. The `os.environ.setdefault("HF_TOKEN", "test-token")` line at the top satisfies T1's gated-gate when present.

### File 3: `pyproject.toml`

Generated by `uv init`. Dependencies are added by Step 4.

### File 4: `.env.example`

```
# Hugging Face access token. Required for gated datasets only.
# Get one at https://huggingface.co/settings/tokens.
HF_TOKEN=

# Maximum number of rows to load from the dataset.
# Higher values use more memory; lower values speed up startup.
MAX_ROWS=10000
```

## Step 4: Code quality

```bash
pip install uv --break-system-packages

uv init --name dash-app
uv add dash dash-bootstrap-components plotly pandas python-dotenv datasets huggingface-hub
uv add --dev ruff ty pytest

uv run ruff check --fix dash_app.py test_dash_app.py
uv run ruff format dash_app.py test_dash_app.py
uv run ty check dash_app.py
```

Fix any error before delivering.

## Step 5: Testing

```bash
uv run pytest test_dash_app.py -v
```

Fix any failure before delivering.
